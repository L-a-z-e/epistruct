# Notes MD

대화에서 설명한 내용을 `.claude/notes/`에 마크다운 파일로 저장한다.

## 실행 순서

### Step 1 — 저장 범위 확인 (사용자 응답 대기)

먼저 사용자에게 무엇을 저장할지 확인한다. 바로 저장하지 않는다.

아래 형식으로 물어본다:

```
다음 내용을 저장할까요?

📄 파일명: {제안 파일명}.md
📌 주제: {주제 한 줄}
📝 포함 내용:
  - {저장할 섹션 1}
  - {저장할 섹션 2}
  - ...

범위를 조정하거나 파일명을 바꾸고 싶으면 말해줘요.
```

사용자가 확인("저장해", "ㅇㅇ", "진행해" 등) 하면 Step 2로 넘어간다.
범위나 파일명 수정을 요청하면 반영 후 재확인한다.

### Step 2 — 백그라운드 저장 + 즉시 다음 작업 안내

확인이 완료되면:
1. Agent 툴로 저장 작업을 **백그라운드**로 위임한다
2. 저장 시작을 한 줄로 알리고 즉시 다음 대화로 넘어간다

```
저장 시작했어요 → .claude/notes/{파일명}.md (백그라운드)
```

백그라운드 Agent에게 전달할 내용:
- 저장 경로: `{cwd}/.claude/notes/{파일명}.md`
- 노트 내용 (아래 포맷 기준으로 작성)
- INDEX.md 업데이트 (`{cwd}/.claude/notes/INDEX.md`)

## 노트 파일 포맷

```markdown
---
date: YYYY-MM-DD
tags: [태그1, 태그2]
---

# 제목

## 핵심 개념
(한 줄 요약)

## 상세 설명
(대화에서 설명한 내용 — 요약이 아닌 실제 내용)

## 코드 예시
(있을 경우)

## 관련 개념
(있을 경우)
```

## INDEX.md 포맷

INDEX.md가 없으면 새로 생성, 있으면 테이블에 행 추가.

```markdown
# Notes Index

| 파일 | 주제 | 날짜 |
|------|------|------|
| [파일명.md](파일명.md) | 주제 한 줄 설명 | YYYY-MM-DD |
```

## 규칙

- 저장 경로: `{현재 프로젝트}/.claude/notes/` — 없으면 자동 생성
- 파일명: 소문자 kebab-case, 공백 없음, 영문 기반
- 내용: 대화에서 실제로 설명한 내용 그대로 — 추상적 요약 금지
- 한국어/영어 혼용 유지 — 설명된 언어 그대로
- 같은 주제 파일이 이미 있으면 Step 1 확인 시 사용자에게 알리고 덮어쓸지 확인
