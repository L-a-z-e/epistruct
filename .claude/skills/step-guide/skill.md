# Step Guide 스킬

체크리스트 항목을 상세 단계로 분해하고, 사용자와 하나씩 확인하며 진행한다.
각 단계는 사용자가 명시적으로 승인해야 다음으로 넘어간다.

## 트리거

- "단계별로 진행하자 / 하나씩 확인하면서 진행"
- "가이드 모드로 해줘"
- `/step-guide`

## 핵심 원칙

1. **단계 확인 게이트**: 현재 단계가 완료·승인되기 전까지 다음 단계로 넘어가지 않는다
2. **답변 점층**: 첫 설명은 핵심만 → 사용자가 "왜?", "자세히", "설명해줘" 물으면 그때 상세화
3. **질문 우선**: 사용자가 질문하면 구현보다 설명 먼저 — 이해 확인 후 진행
4. **트레이드오프 제시**: 선택지가 있을 때는 옵션과 트레이드오프를 먼저 제시하고 사용자가 선택

## 실행 흐름

### Phase 1 — 단계 목록 제시

스킬 시작 시 또는 작업 항목이 주어지면:

1. 해당 작업을 3~8개의 구체적 단계로 분해한다
2. 아래 포맷으로 전체 단계 목록을 출력한다:

```
## [작업명] 단계 목록

| # | 단계 | 내용 |
|---|------|------|
| 1 | 단계명 | 한 줄 설명 |
| 2 | 단계명 | 한 줄 설명 |
...

→ 1단계부터 시작할까요?
```

3. 사용자 확인 후 1단계로 진입한다

### Phase 2 — 단계별 진행

각 단계 진입 시:

```
---
## 단계 N/전체: {단계명}

**무엇을 하는가**: 한 두 줄 핵심 설명

{코드 또는 명령어 블록}

질문 있으면 물어보세요. 진행하려면 "진행" 또는 "다음"이라고 해주세요.
```

**단계 내 상태 머신**:

```
[단계 제시] → 사용자 질문 → [간결 답변] → (추가 질문 or 진행)
                                    ↑              |
                                    └──────────────┘
           → 사용자 "진행" → [구현/실행] → [완료 확인] → 다음 단계
```

**질문 유형별 답변 방식**:

| 질문 유형 | 답변 방식 |
|-----------|-----------|
| "이게 뭐야?" / "왜 이렇게?" | 1~3문장 핵심만 |
| "자세히 설명해줘" / "더 알고 싶어" | 개념 + 예시 + 원리 상세화 |
| "다른 방법은?" / "비교해줘" | 옵션 테이블 + 트레이드오프 |
| "문법이 어떻게 돼?" | 코드 블록 + 핵심 포인트 2~3개 |
| 에러 발생 | problem-solving 규칙 적용 — 에러 전문 분석 후 원인 설명 |

### Phase 3 — 단계 완료 처리

각 단계 구현 완료 후:

1. 실행 결과 또는 변경 파일 요약 출력
2. "이 단계 완료됐습니다. 다음 단계(N+1)로 넘어갈까요?" 확인
3. 사용자 승인 → 다음 단계 진입
4. 사용자 "잠깐" / 추가 질문 → 답변 후 다시 확인 대기

### Phase 4 — 전체 완료

모든 단계 완료 시:

```
## 완료: {작업명}

완료된 단계:
- [x] 단계1
- [x] 단계2
...

체크리스트 항목 업데이트할까요?
```

→ 승인 시 `docs/planning/CHECKLIST.md`에서 해당 항목 `[ ]` → `[x]` 변경

## 출력 규칙

- 단계 제시 시 코드/명령어는 반드시 블록으로 감싼다
- 설명은 짧게 — 사용자가 더 요청할 때만 늘린다
- 다음 단계 번호와 이름을 항상 명시한다
- 에러 발생 시 즉시 원인 분석 모드로 전환 (구현 중단)
