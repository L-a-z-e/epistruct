# 노드 추출 리뷰 UX 플로우

> 버전: v1.0  
> 작성일: 2026-06-02  
> 관련 API: `docs/design/api-spec.md` — ai_pipeline, knowledge(GraphQL)

---

## 1. 개요

소스 제출 후 AI가 추출한 draft 노드를 사용자가 직접 검토·확정하는 흐름.  
핵심 원칙: **사람이 인지하고 통제한다** — AI 결과를 그대로 수락하지 않고, 읽고 수정하고 판단한다.

---

## 2. 전체 화면 흐름

```
[입력 화면]
    │
    ▼  POST /api/v1/pipeline/sources → 202 Accepted
[처리 중 화면]  ◄── SSE /api/v1/pipeline/jobs/:id/stream
    │
    ├── succeeded → 자동 전환
    ▼
[리뷰 화면]  ◄── 미처리 draft 존재 시 언제든 재진입 가능
    │
    ├── 개별 수락/거부
    ├── 인라인 편집 → 수락
    ├── 전체 수락
    └── 선택 일괄 수락/거부
    │
    ▼  모든 draft 처리 완료 시
[그래프 화면]  (confirmed 노드 반영)
```

## 3. 처리 중 화면

### 3.1 레이아웃

```
┌─────────────────────────────────┐
│  소스 제목 또는 URL               │
│                                 │
│       ⊙  분석 중...              │
│                                 │
│  청킹 완료                  ✓   │
│  노드 추출 중...           ···   │
│  임베딩 생성               ○   │
│                                 │
│  [취소]                         │
└─────────────────────────────────┘
```

### 3.2 SSE 이벤트 → UI 매핑

| SSE event | 화면 상태 |
|-----------|---------|
| `chunking` | "청킹 중..." 스피너 |
| `extracting` | "노드 추출 중..." 스피너 |
| `nodes_extracted: N` | "N개 노드 발견됨" 업데이트 |
| `embedding` | "임베딩 생성 중..." 스피너 |
| `succeeded` | 리뷰 화면으로 자동 이동 |
| `failed` | 에러 화면 (→ 3.3) |

### 3.3 실패 화면

```
추출에 실패했습니다
사유: [에러 메시지]

[다시 시도]  [돌아가기]
```

```
│                                         │

│  │                                 │   │

│  ┌─────────────────────────────────┐   │
│  │ [M] React.memo          [☐]    │   │

│  ┌─────────────────────────────────┐   │
│  │ [C] useCallback    ✓ 확정됨     │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

### 4.2 노드 카드 상태

| 상태 | 스타일 | 액션 |
|------|--------|------|
| `draft` (미처리) | 기본 카드 | [편집] [수락] [거부] |
| `confirmed` | 초록 테두리 + "확정됨" | — (읽기 전용) |
| `rejected` | 회색 + 취소선 + "거부됨" | — (읽기 전용) |

확정/거부된 노드는 목록 하단 "처리 완료" 섹션으로 이동.

### 4.3 node_type 뱃지

| 타입 | 뱃지 | 색상 |
|------|------|------|
| `P` (Principle) | `[P]` | 보라 |
| `C` (Concept) | `[C]` | 파랑 |
| `M` (Manifestation) | `[M]` | 초록 |

## 5. 상세 플로우

### 5.1 개별 수락

```
수락 버튼 클릭
  → confirmNode(id) Mutation
  → 성공: 카드 → "확정됨" 상태, 하단 이동
  → 실패(NODE_STATUS_CONFLICT): 토스트 "이미 처리된 노드입니다", 상태 새로고침
```

### 5.2 인라인 편집 → 수락

```
편집 버튼 클릭
  → 카드 내 label/description 텍스트 필드 활성화

┌─────────────────────────────────┐
│ [C] [label 편집 필드       ]    │
│ [description 편집 필드         ] │
│ [편집 필드                     ] │
│                                 │
│        [취소]  [저장 후 수락]    │
└─────────────────────────────────┘

저장 후 수락 클릭
  → updateNode(id, {label, description}) Mutation
  → 성공: confirmNode(id) Mutation
  → 최종 성공: 카드 → "확정됨" 상태
```

편집 후 수락하지 않고 저장만 원하는 경우는 Phase 1-A 범위 밖 — 편집은 항상 수락과 함께.

### 5.3 개별 거부

```
거부 버튼 클릭
  → 확인 없이 즉시 rejectNode(id) Mutation
  → 성공: 카드 → "거부됨" 상태, 하단 이동
```

거부는 가역적이지 않음 (API 레벨 `NODE_STATUS_CONFLICT`). 잘못 거부한 경우 → F4 편집 기능으로 처리 (Phase 1-A 구현 예정).

### 5.4 전체 수락

```
[모두 수락] 버튼 클릭
  → 확인 모달: "N개 노드를 모두 수락합니다. AI 추출 결과를 그대로 반영합니다."
              [취소]  [수락]

수락 확인
  → draft 노드 전체에 confirmNode 순차 실행
  → 진행 표시: "수락 중... (3/12)"
  → 완료: 모든 카드 "확정됨"
```

### 5.5 선택 일괄 처리

```
카드 우측 상단 체크박스 선택
  → 헤더 액션 바: [선택 수락 (N)]  [선택 거부 (N)]

선택 수락 클릭
  → 선택된 노드에 confirmNode 순차 실행
  → 완료: 처리된 카드 상태 업데이트, 체크박스 초기화

선택 거부 클릭
  → 확인 없이 선택된 노드에 rejectNode 순차 실행
```

### 5.6 부분 처리 후 이탈

```
리뷰 화면에서 뒤로 가기 / 앱 종료
  → 미처리 draft 노드는 DB에 그대로 유지

재진입 경로:
  - 홈/그래프 화면에 "미처리 노드 X개" 뱃지
  - 뱃지 탭 → 리뷰 화면 (해당 space의 draft 노드 목록)
```

---

## 6. 완료 후 전환

```
모든 draft 노드 처리 완료 (confirmed + rejected)
  → 토스트: "X개 노드가 그래프에 추가되었습니다"
  → 3초 후 자동으로 그래프 화면 이동
     또는 [그래프 보기] 버튼으로 즉시 이동
```

## 7. API 연동 매핑

| UX 액션 | API |
|---------|-----|
| 리뷰 화면 진입 | `nodes(spaceId, filter:{status:draft}, first:20)` GraphQL Query |
| 추가 로드 | `nodes(..., after: cursor)` — cursor 페이지네이션 |
| 수락 | `confirmNode(id: ID!)` Mutation |
| 거부 | `rejectNode(id: ID!)` Mutation |
| 편집 | `updateNode(id, patch:{label, description})` Mutation |
| 처리 중 진행 | `GET /api/v1/pipeline/jobs/:id/stream` SSE |
| 처리 중 폴링 (SSE 불가 시) | `GET /api/v1/pipeline/jobs/:id` |

## 8. 엣지 케이스

| 상황 | 처리 |
|------|------|
| 노드 0개 추출 | "추출된 노드가 없습니다. 소스를 다시 확인해보세요." + [재제출] |
| 확정/거부 API 실패 | 토스트 에러 + 카드 상태 롤백 |
| 네트워크 끊김 중 일괄 처리 | 실패한 항목만 에러 표시, 성공한 항목은 유지 |
| 같은 소스 중복 제출 | 파이프라인에서 `PIPELINE_DUPLICATE_SOURCE` 409 반환 (리뷰 화면 진입 전 차단) |
| P(Principle) 타입 추출 시 | Phase 1-A에서는 C/M으로만 추출. P 노드는 Phase 1-B에서 별도 확정 게이트 추가 예정 |
